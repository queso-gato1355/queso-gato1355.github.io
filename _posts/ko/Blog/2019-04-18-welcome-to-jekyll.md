이 게시물은 `_posts` 디렉토리에서 찾을 수 있습니다. 편집하고 사이트를 다시 빌드하여 변경 사항을 확인하세요. 사이트를 다시 빌드하는 방법은 여러 가지가 있지만, 가장 일반적인 방법은 `jekyll serve`를 실행하는 것입니다. 이는 웹 서버를 시작하고 파일이 업데이트될 때 사이트를 자동으로 다시 생성합니다.

새 게시물을 추가하려면 `_posts` 디렉토리에 `YYYY-MM-DD-name-of-post.ext` 규칙을 따르는 파일을 추가하고 필요한 front matter를 포함하세요. 이 게시물의 소스를 살펴보면 어떻게 작동하는지 알 수 있습니다.

Jekyll은 코드 스니펫에 대한 강력한 지원도 제공합니다:

```ruby
def print_hi(name)
  puts "Hi, #{name}"
end
print_hi('Tom')
#=> prints 'Hi, Tom' to STDOUT.
```

Jekyll을 최대한 활용하는 방법에 대한 자세한 내용은 [Jekyll 문서][jekyll-docs]를 확인하세요. 모든 버그/기능 요청은 [Jekyll의 GitHub 저장소][jekyll-gh]에 제출하세요. 질문이 있으시면 [Jekyll Talk][jekyll-talk]에서 물어볼 수 있습니다.
